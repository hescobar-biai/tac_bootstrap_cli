# Load AI Docs Command Template

**ADW ID:** feature_Tac_9_task_21
**Date:** 2026-01-26
**Specification:** specs/issue-262-adw-feature_Tac_9_task_21-sdlc_planner-load-ai-docs-command.md

## Overview

Created Jinja2 template for the `/load_ai_docs` slash command that enables loading AI documentation into Claude Code context via specialized exploration agents. This command allows agents to efficiently ingest TAC methodology documentation from the `ai_docs/doc/` directory to understand project-specific AI patterns and guidelines.

## What Was Built

- **Jinja2 Template**: Created `load_ai_docs.md.j2` template with minimal parameterization
- **Rendered Example**: Generated `.claude/commands/load_ai_docs.md` as reference implementation
- **Documentation Loading**: Command uses Task tool with Explore agent for efficient documentation scanning
- **Filtering Support**: Supports loading all docs or filtering by document number (e.g., "1-3" for TAC courses 1-3)

## Technical Implementation

### Files Modified

- `tac_bootstrap_cli/tac_bootstrap/templates/claude/commands/load_ai_docs.md.j2`: Jinja2 template for generating the load_ai_docs command in target projects
- `.claude/commands/load_ai_docs.md`: Rendered example for tac_bootstrap repo demonstrating proper template output

### Key Changes

- Template structure follows existing command patterns (Variables, Instructions, Run, Examples, Report sections)
- Hardcoded documentation path as `ai_docs/doc/` (no project-specific parameterization needed)
- Integrated Task tool with `subagent_type=Explore` for efficient large documentation scanning
- Implemented filtering syntax supporting single docs ("5"), ranges ("1-3"), and multiple ranges ("1-3,5,7-8")
- Added comprehensive examples showing various usage patterns (load all, filter by range, handle errors)
- Included detailed reporting format for loaded documents, key topics, and missing files

## How to Use

### For TAC Bootstrap Developers

1. Template is automatically available in `templates/claude/commands/` directory
2. When generating a new project, the template renders into `.claude/commands/load_ai_docs.md`
3. No configuration needed - template uses sensible defaults

### For Generated Project Users

In a project generated by TAC Bootstrap:

1. Load all AI documentation:
   ```
   /load_ai_docs
   ```

2. Load specific TAC courses (e.g., courses 1-3):
   ```
   /load_ai_docs doc_filter=1-3
   ```

3. Load single document:
   ```
   /load_ai_docs doc_filter=5
   ```

4. Load multiple ranges:
   ```
   /load_ai_docs doc_filter=1-2,5,7-8
   ```

## Configuration

- **Documentation Path**: Hardcoded as `ai_docs/doc/` (no config override)
- **Expected Structure**: TAC courses numbered 1-8 (doc/1.md through doc/8.md)
- **Agent Configuration**: Uses Explore agent with "medium" thoroughness for balanced scanning
- **No Dependencies**: Works with existing TAC Bootstrap infrastructure
